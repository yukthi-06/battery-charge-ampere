# Implementation Plan - Freeze Views and Timer on Disconnect

## Problem
When the charger is disconnected, the duration timer and the average current calculation should freeze. Only the real-time charging/discharging current should continue to update. In addition, the visual appearance of the frozen fields (Duration and Average) should change (e.g. to a muted color) to indicate their frozen status.

## Proposed Changes
Modify the lifecycle and periodic update logic in `MainActivity.java`:
- If charger is **connected**:
  - Sample current using `recordSample()` (registers a new sample into in-memory average).
  - Resume/Start the duration timer (`timerManager.start()`).
  - Restore standard primary and secondary colors of the `Duration` and `Average` text views.
- If charger is **disconnected**:
  - Sample current using `getCurrentNow()` (queries real-time value without adding it to the samples, freezing the average calculation).
  - Stop/Pause the duration timer (`timerManager.stop()`).
  - Set text color of the `Duration` and `Average` text views to a muted gray (`@color/colorDisconnected`).
