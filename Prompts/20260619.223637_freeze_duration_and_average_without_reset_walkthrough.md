# Walkthrough - Freeze Duration and Average without Reset

I have updated the disconnect broadcast handling behavior to keep the duration and average values frozen on the screen.

## Changes Made
### Activities
Modified `powerConnectionReceiver` in [MainActivity.java](file:///x:/Projects_X/0_Active/8_Android_APK/05_Early_Stage/Battery_Charge_Ampere_APK_GITHUB/app/src/main/java/com/vypeensoft/chargeampmeter/MainActivity.java) to only call `resetMeasurementSession()` upon connection. Disconnection now calls `performSampleUpdate()` which freezes the views and locks the values in place without resetting them.

## Verification Results
- Connected: Telemetry counts up, average recalculates, text color is normal.
- Disconnected: Telemetry duration and average values freeze in place (holding their last calculated values rather than resetting to zero), and text color shifts to muted gray.
