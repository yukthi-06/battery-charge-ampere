# User Prompt
Even when the charger is disconnected, the current charging is showing 500mA or some such values and it keep changing in that range

---

# Implementation Plan - Fix Disconnect Charging Current Representation

## Problem
The app displayed positive discharging current values (e.g., 500 mA) when the charger was disconnected because the normalization logic unconditionally applied `Math.abs()` to the telemetry result.

## Proposed Changes
Modify the normalization helper `normalizeCurrent` to check the charger status:
- If **connected**, return positive current values (charging).
- If **disconnected**, return negative current values (discharging).
