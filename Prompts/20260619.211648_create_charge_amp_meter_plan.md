# User Prompt
Create a complete Android application in Java (NOT Kotlin) called "Charge Amp Meter".

## Purpose

The application should detect when a charger is connected and continuously measure and display the battery charging current in amperes (A) and milliamperes (mA) using Android's BatteryManager APIs.

## Main Requirements

### Main Screen

Display the following information:

1. Current Charging Value

   * Real-time charging current.
   * Show in mA and A.
   * Update every second.

2. Average Charging Value

   * Calculate the average charging current from the beginning of the measurement session.
   * Show in mA and A.

3. Measurement Duration

   * Display elapsed measurement time in:

     * Hours
     * Minutes
     * Seconds

4. Charger Status

   * Connected
   * Disconnected
   * Charging
   * Not Charging

5. Reset Button

   * A button labeled "Reset Measurement".
   * When pressed:

     * Reset average calculations.
     * Reset timer.
     * Clear all collected samples.
     * Start a fresh measurement session immediately.

### Charging Current Measurement

Use:

* BatteryManager.BATTERY_PROPERTY_CURRENT_NOW
* BatteryManager.BATTERY_PROPERTY_CURRENT_AVERAGE (when available)

Implement device compatibility handling because some devices return:

* Positive values
* Negative values
* Microamps instead of milliamps

Normalize values and display correctly.

### Data Sampling

* Sample current every 1 second.
* Store samples in memory.
* Average value should be calculated from all samples collected since the last reset.

Formula:

Average Current =
(Sum of all sampled currents) / (Number of samples)

# Implementation Plan: Charge Amp Meter App

We will build the **Charge Amp Meter** Android application in pure Java. The project will feature a modern, clean, and dynamic design utilizing Google's Material Components and AppCompat APIs.

## Proposed Architecture

1. **Activities**:
   - `MainActivity`: Hosts the main screen with Navigation Drawer, real-time measurements, average values, duration, charger status, and the Reset button.
   - `SettingsActivity`: Manages app settings (Sampling Interval, Keep Screen On, Display Unit, Dark Mode) and persists them using `PreferenceManager`.
   - `HelpActivity`: Explains charging parameters, compatibility differences, and how charging current measurements work.
   - `AboutActivity`: Shows application metadata, description, and developer info.

2. **Helpers**:
   - `PreferenceManager`: Handles reading and writing user preferences to `SharedPreferences` in a centralized class.
   - `TimerManager`: A runnable/handler-based timer that manages measurement sessions, updates elapsed time, and coordinates periodic sampling.
   - `ChargingMonitor`: Handles interacting with the `BatteryManager` APIs, reading the current, normalizing differences (microamps vs. milliamps, positive vs. negative depending on status), and managing samples.

## UI Design & Themes

We will define custom theme styles using Material Components:
- **Primary Color**: Vibrant teal/cyan `#00BCD4` (glowing cyan for active state) and `#00E676` (green for charging success).
- **Surface Colors**: Deep gray background for Dark Mode, clean light grey/white for Light Mode.
- **Card Designs**: Four distinct cards:
  1. *Current Charging Value Card*: Large readable layout with a gauge-like progress border/state representing active current flow.
  2. *Average Charging Value Card*: Clean numerical readout.
  3. *Duration Card*: Timer showing `HH:MM:SS`.
  4. *Charger Status Card*: Text status ("Connected (AC)", "Disconnected", etc.) with a color-coded icon.
